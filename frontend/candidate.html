TechnicalQ1 / 12
Explain how React's reconciliation algorithm works. How does the virtual DOM differ from the real DOM, and what optimization strategies would you use for a large list?
React's reconciliation uses a diffing algorithm comparing the new VDOM tree with the previous using O(n) heuristics: same-type elements keep their state; different-type elements are destroyed. Keys identify list items. For large lists: use React.memo, virtualization (react-window), stable keys, and avoid inline function/object creation inside render.
TechnicalQ2 / 12
Walk me through how you'd diagnose and fix a slow React application. What tools do you use and in what order?
Start with Chrome DevTools Performance tab to profile runtime. Use React DevTools Profiler to find unnecessary re-renders. Apply useMemo, useCallback, React.memo. Analyze bundle size with webpack-bundle-analyzer. Implement code-splitting with React.lazy.
